# MPStats Workflow: пользовательская документация

Эта инструкция описывает проект с точки зрения пользователя: что он делает, как запускать выгрузки, что означает каждый раздел интерфейса, как считаются итоговые показатели и где искать результат.

Документ намеренно не уходит в технические детали кода. Здесь важны рабочие сценарии, правила расчётов и смысл экранов.

## 1. Что делает проект

Проект помогает собрать данные из MPStats и довести их до удобного аналитического результата.

Основной поток такой:

1. Вы выбираете маркетплейсы, категории и месяцы.
2. Приложение скачивает отчёты из MPStats.
3. Данные приводятся к единому виду, даже если у разных маркетплейсов разные названия колонок.
4. Из названий товаров определяется вес или объём упаковки.
5. Считаются объём продаж в кг/тоннах и цена за кг.
6. Товары классифицируются по пользовательским правилам.
7. Результат сохраняется в файлы и в локальный куб/БД для поиска и повторного использования.

Проект не меняет данные в MPStats. Он только скачивает отчёты, обрабатывает их локально и сохраняет результаты на вашем компьютере.

## 2. Основные понятия

**Проект**

Название выгрузки или направления работы. По нему приложение разделяет файлы, планы загрузки и записи в локальной БД. Например: `Мясо апрель`, `Мыло 2026`, `Сахар анализ`.

**Cookie MPStats**

Доступ к вашему текущему сеансу MPStats. Нужен только для скачивания отчётов. Если cookie устарел, скачивание начнёт падать с ошибкой доступа. Cookie хранится только локально в настройках web-приложения и локальной DuckDB.

**Категория**

Человекочитаемая группа товаров: например, `Мясо`, `Мыло`, `Сахар`.

**Путь MPStats**

Путь категории внутри MPStats. Именно по нему приложение понимает, какой раздел MPStats скачать.

**Фильтр**

Дополнительное ограничение внутри категории. Например, скачать только товары, где название содержит `жидкое`, или исключить товары, где название содержит `хозяйственное`.

**Умный план**

Список задач, который приложение создаёт перед запуском и сверяет с локальными файлами. Одна задача обычно равна связке:

```text
маркетплейс + категория/путь + год + месяц
```

**Куб**

Сохранённый набор обработанных данных по месяцам, маркетплейсам и категориям. В разделе `Данные` -> `Куб` можно увидеть, какие месячные срезы уже есть, и искать товары.

**Классификатор**

Набор правил, которые заполняют или исправляют колонки вроде `Подкатегория`, `Тип`, `Вид мяса`, `Бренд`.

## 3. Как запускать web-приложение

Основной пользовательский способ работы на macOS:

```text
MPStats Local App.command
```

Дважды кликните по этому файлу в корне проекта. Откроется локальное приложение в браузере.

Пока открыто окно запуска, приложение работает. Чтобы остановить приложение, закройте окно запуска или нажмите `Ctrl+C` в нём.

На Windows используйте:

```text
MPStats Local App.bat
```

Этот запускатель не требует Node.js и админских прав. При первом запуске он создаёт локальную `.venv`, устанавливает Python-зависимости внутрь проекта и запускает приложение из готовой сборки `web/dist`. Если `web/dist` отсутствует, приложение на Windows не стартует: frontend нужно заранее собрать на компьютере, где есть Node.js, или взять уже подготовленную папку из проекта.

Все пользовательские действия выполняются через web-интерфейс. Старые отдельные CLI, notebook и desktop-GUI сценарии больше не поддерживаются.

## 4. Рекомендуемый рабочий сценарий

### Историческая загрузка

Используйте этот режим, когда нужно скачать диапазон месяцев: например, с января по апрель.

1. Откройте приложение.
2. Выберите режим `Историческая загрузка`.
3. Откройте `Проекты`, нажмите `Создать` и задайте имя проекта, если нужного проекта ещё нет.
4. Вставьте актуальный `MPStats cookie`.
5. Откройте вкладку `Категории`.
6. Отметьте нужные категории/пути.
7. Откройте `Умный план` и укажите период: с какого года/месяца по какой год/месяц.
8. Нажмите `Создать план`.
9. Проверьте вкладку `Умный план`.
10. Нажмите `Запустить`.
11. После завершения проверьте `Данные` -> `Куб` и `Данные` -> `Файлы`.

### Ежемесячное обновление

Используйте этот режим, когда уже есть сохранённый куб и нужно подтянуть следующий месяц.

1. Выберите режим `Ежемесячное обновление`.
2. Проверьте текущий проект в левом списке.
3. Вставьте актуальный cookie, если он устарел.
4. Нажмите `Синхронизировать новый месяц`.

Приложение смотрит, какой месяц уже сохранён последним, и создаёт план на следующий месяц. Если сохранённых месяцев ещё нет, берётся текущий месяц.

## 5. Разделы интерфейса

### Верхняя панель

Вверху выбирается режим работы:

- `Историческая загрузка` — ручной диапазон месяцев;
- `Ежемесячное обновление` — следующий месяц после последнего сохранённого;
- `Инструкция` — встроенная короткая справка внутри приложения.

В этой же панели есть авторская подпись `Made by @exoldoff` со ссылкой на GitHub и отдельная иконка Telegram.

В окне `Инструкция` есть кнопка `Полная инструкция`. Она открывает этот markdown-документ из проекта в отдельной вкладке браузера.

Основная навигация в центре оставляет только крупные разделы: `Проекты`, `Умный план`, `Категории`, `Данные`, `Классификатор`. Внутри раздела `Данные` отдельной короткой строкой выбираются `Куб`, `Отчёты`, `Файлы`, `Выгрузка` и `Качество`.

### Левая колонка: проект и настройки

Левую колонку можно свернуть кнопкой с иконкой панели. Это удобно после выбора проекта и cookie: центральный рабочий экран становится шире, а настройки не остаются постоянно перед глазами.

#### Проект и доступ

Здесь задаются:

- `Текущий проект` — выбор из уже созданных проектов;
- `MPStats cookie` — доступ для скачивания отчётов.

Новый проект создаётся только в разделе `Проекты` кнопкой `Создать`. После создания он появляется в списке слева и становится текущим.

Рядом с названиями полей есть значок `?`. Наведите на него курсор, чтобы увидеть, что именно меняет поле и когда его стоит использовать.

Кнопка `Сохранить настройки` сохраняет эти значения локально.

#### Настройки pipeline

Эти настройки управляют повторными запусками. По умолчанию блок компактный: видны только ключевые значения. Нажмите `Открыть`, чтобы изменить полный набор параметров.

У каждого параметра есть подсказка `?` рядом с названием. Она объясняет эффект настройки, безопасное значение по умолчанию и случаи, когда параметр стоит менять.

**Перескачивать raw**

Если выключено, приложение не скачивает заново файл, который уже есть. Если включено, raw-файл будет скачан повторно из MPStats.

**Пересобирать processed**

Если выключено, готовый обработанный файл используется повторно. Если включено, приложение заново пересчитает обработанный файл из raw.

**Перезаписывать БД**

Если выключено, уже сохранённый срез в кубе не дублируется. Если включено, приложение заменит данные для той же связки проект + месяц + маркетплейс + категория.

**Количество повторов**

Сколько раз повторить скачивание, если MPStats временно не ответил или отчёт не успел подготовиться.

**Таймаут, сек**

Сколько ждать подготовки отчёта MPStats.

**Пауза между запросами**

Пауза между скачиваниями. Обычно лучше не ставить ноль, чтобы не упираться в ограничения MPStats.

**Параллельные скачивания**

Настройка для режима загрузки. В обычной работе безопаснее оставлять `1`.

**Максимальный вес, кг**

Порог, после которого найденный вес считается подозрительным. По умолчанию `40 кг`. Всё, что выше, помечается как аномалия, если приложение не смогло исправить очевидную ошибку формата.

### Правая колонка: текущий запуск

Здесь видно состояние последнего плана:

- период;
- количество категорий;
- количество месяцев;
- всего задач;
- сколько готово;
- сколько ошибок;
- сколько осталось;
- текущий шаг.

Основные кнопки:

**Создать план**

Создаёт список задач, но ещё не скачивает данные. В историческом режиме кнопка активна только после выбора категорий и корректного периода.

**Запустить**

Запускает ожидающие задачи и задачи с ошибками. Недоступна, если плана нет, задач нет или запуск уже идёт.

**Пауза**

Мягко останавливает выполнение между стадиями. Активна только во время выполнения. Уже начатая стадия должна завершиться, после этого план перейдёт в `пауза`, а незавершённые задачи останутся в плане.

**Остановить**

Останавливает текущий pipeline и переводит план в `остановлено`. Уже завершённые стадии и файлы остаются на месте, незавершённые задачи не продолжаются автоматически. Если потом нужно догрузить остаток, нажмите обычный `Запустить` для этого плана.

**Продолжить**

Продолжает незавершённый запуск. Активна только для плана на паузе.

**Повторить ошибки**

Запускает только задачи со статусом ошибки. Активна только если ошибки действительно есть.

**Собрать куб из готовых файлов**

Использует уже готовые обработанные/классифицированные файлы и сохраняет их в БД без повторного скачивания.

**Переклассифицировать куб**

Заново применяет текущие правила классификатора к processed-файлам выбранного плана, перезаписывает classified-файлы и заменяет соответствующие срезы в локальном кубе/БД.

**Синхронизировать новый месяц**

Создаёт и запускает план на следующий месяц после последнего сохранённого месяца.

### Вкладка `Умный план`

Для исторической загрузки период выбирается именно здесь, перед созданием плана. Это не настройка проекта: один и тот же проект можно догружать разными периодами.

В режиме `Ежемесячное обновление` год и месяц не выбираются вручную. Приложение смотрит registry куба и берёт следующий месяц автоматически.

### Последние планы

Список последних созданных планов. Можно открыть старый план, посмотреть задачи, ошибки и продолжить работу.

Кнопка с корзиной удаляет сам план и его задачи из локальной DuckDB. Файлы проекта и уже сохранённые срезы куба при этом не удаляются. Выполняющийся план сначала нужно поставить на паузу или дождаться завершения.

## 6. Вкладка `Категории`

На этой вкладке выбираются категории для плана загрузки.

Категории подтягиваются из справочника категорий. Они сгруппированы по названию категории, внутри группы могут быть разные маркетплейсы, пути, периоды и фильтры.

В строке категории видно:

- маркетплейс;
- путь MPStats;
- признак `FBS`, если он включён;
- период активности, если он задан.

Можно выбрать:

- отдельный путь;
- всю группу категории кнопкой на заголовке группы;
- несколько категорий сразу.

Поиск ищет по названию категории, маркетплейсу, пути и периоду.

## 6.1. Табличные фильтры и сортировки

В таблицах приложения заголовок каждого столбца открывает меню как в Excel:

- `А-Я` / `Я-А` сортирует строки по выбранному столбцу;
- поле `Текст` оставляет строки, где значение столбца содержит введённую фразу;
- блок `Список` позволяет выбрать конкретные значения через чекбоксы;
- `Очистить` сбрасывает фильтр текущего столбца;
- `Сбросить` над таблицей убирает все фильтры и сортировку.

Эти фильтры работают в предпросмотрах и таблицах интерфейса: умный план, файлы, куб, предпросмотр БД, предпросмотр выгрузки, справочник и классификатор.

## 7. Редактор `Справочник`

Справочник — это главный список категорий и путей MPStats. Приложение использует CSV-файл:

```text
Справочник категорий MP STATS.csv
```

Excel-дубликаты не считаются источником правды. Если нужно менять категории, откройте `Справочник` в левом блоке `Правила и справочник` или правьте этот CSV.

### Поля справочника

**Активна**

Если выключено, строка не участвует в выборе категорий.

**Категория**

Человекочитаемое название категории. Оно потом попадает в итоговые данные.

**Маркетплейс**

Один из поддерживаемых маркетплейсов: Ozon, WB, Яндекс.Маркет.

**FBS**

Признак FBS для выгрузки. Обычно важен для Ozon.

**От / До**

Период активности строки. Если задано `От`, категория будет участвовать только начиная с этого месяца. Если задано `До`, только до этого месяца включительно.

Примеры:

- `2026` — весь 2026 год;
- `2026-03` — март 2026;
- `03.2026` — март 2026;
- `март 2026` — март 2026.

**Путь**

Основной путь MPStats.

**Фильтр по названию**

Дополнительное ограничение внутри пути. Можно задать одно или два условия:

- `Содержит` — оставить товары, где название содержит фразу;
- `Исключает` — исключить товары, где название содержит фразу.

Если условий два, между ними выбирается связь:

- `И` — должны выполниться оба условия;
- `ИЛИ` — достаточно одного условия.

**Комментарий**

Поле для рабочих заметок.

### Как добавить новую категорию

1. Откройте `Справочник` в левом блоке `Правила и справочник`.
2. Нажмите `Добавить`.
3. Заполните категорию, маркетплейс, период, путь и фильтр.
4. Нажмите `Сохранить CSV`.
5. Перейдите в раздел `Категории`.
6. Нажмите `Обновить справочник`, если новая строка ещё не появилась.

### Как временно отключить категорию

1. Откройте строку в редакторе `Справочник`.
2. Выключите `Активна в приложении`.
3. Нажмите `Сохранить CSV`.

После этого строка останется в справочнике, но не будет участвовать в новых планах.

## 8. Как создаётся умный план

План строится из выбранных категорий и периода.

Для каждой активной выбранной строки справочника приложение берёт все месяцы из выбранного диапазона. Если у строки есть `От` или `До`, месяцы вне этого периода пропускаются.

Итоговая задача создаётся для каждой комбинации:

```text
категория/путь + маркетплейс + месяц
```

Например, если выбрано 3 пути и период 4 месяца, будет до 12 задач. Если у части путей ограничен период активности, задач будет меньше.

Если для строки задан фильтр, он применяется при скачивании из MPStats. Если включён FBS, он тоже учитывается при запросе.

## 9. Вкладка `Умный план`

Это основная вкладка приложения при открытии web-app. Она показывает задачи текущего плана, сверяет их с локальными файлами проекта и отдельно показывает матрицу наполнения куба.

В верхней части есть:

- сводка по статусам;
- рекомендованное действие;
- матрица куба по категориям и маркетплейсам;
- фильтры `Все`, `Готовые`, `Нет файлов`, `Устарели`, `Ошибки`, `Неполные`.

### Матрица куба

Матрица строится по уже сохранённым срезам из `Данные` -> `Куб`.

Строки — категории, колонки — маркетплейсы. В колонке маркетплейса эталоном считается последний месяц, который уже есть в кубе для этого маркетплейса. Ячейка показывает:

- `✅ ММ.ГГГГ`, если для категории есть актуальный срез за последний месяц этого маркетплейса;
- `❌ ММ.ГГГГ`, если срез есть, но он отстаёт от последнего месяца маркетплейса;
- `❌ нет данных`, если для категории и маркетплейса в кубе нет ни одного среза.

Так сразу видно, какие связки категория + маркетплейс надо доскачать или пересобрать.

### Тяжёлые категории и отчёты

Если категория или отдельный месячный срез становится слишком большим, приложение помечает его как `тяжёлая`. Сейчас ориентиры такие:

- срез от 250 000 строк;
- категория от 1 000 000 строк суммарно по проекту.

Тяжёлый режим не удаляет raw/processed/classified-файлы и не меняет расчёты. Он меняет рекомендуемый способ анализа: вместо открытия миллионов сырых строк в Excel используйте `Данные` -> `Отчёты`.

Во вкладке `Куб` у таких срезов появляется режим `тяжёлая` и причина. Во вкладке `Отчёты` видно список тяжёлых категорий, доступные отчётные таблицы и время последней сборки отчёта.

Доступные отчёты:

- `Категория по месяцам` — продажи, выручка, объём и цены по категории, маркетплейсу и месяцу;
- `Бренды по месяцам` — те же метрики с разрезом по брендам;
- `Классификация по месяцам` — разрез по колонкам классификатора, если они есть в кубе;
- `Топ SKU` — агрегированный топ товаров по выручке и продажам без выгрузки raw-строк.

Excel-файлы из `Отчёты` уже подготовлены для анализа и сводных таблиц. Это основной способ работать с большими категориями.

Умный план сравнивает желаемые задачи `маркетплейс + категория + месяц` с рабочими файлами:

- `raw`;
- `processed`;
- `classified`;
- сохранённым срезом в `Данные` -> `Куб`.

В таблице есть стадии:

**Скачивание**

Получение raw-файла из MPStats.

**Обработка**

Приведение колонок к единому виду, расчёт веса, объёма и цены за кг.

**Классификация**

Применение правил классификатора.

**БД**

Сохранение результата в локальный куб.

### Статусы умного плана

**готово**

Есть актуальный classified-файл или срез уже сохранён в `Данные` -> `Куб`.

**нет файлов**

Для задачи пока нет локальных файлов. Обычно нужно запустить скачивание.

**устарело**

Один из upstream-файлов новее результата: например, `raw` новее `processed`, `processed` новее `classified` или `classified` новее сохранённого среза БД.

**ошибка**

Последний запуск задачи упал. Текст ошибки видно в строке.

**неполно**

Есть только часть файлов: например, `raw` уже скачан, но `processed` или `classified` ещё не собраны.

### Стадии pipeline

**ожидает**

Задача создана, но ещё не выполнялась.

**скачивание / скачано**

Файл скачивается или уже скачан.

**обработка / обработано**

Файл обрабатывается или уже обработан.

**классификация / классиф.**

Применяются правила классификатора или результат уже готов.

**сохранение / в БД**

Данные сохраняются или уже сохранены в куб.

**ошибка**

Задача упала. Текст ошибки видно в строке.

**нет данных**

MPStats вернул пустой файл или в нём нет строк.

**пропуск**

Приложение не выполняло стадию, потому что нужный файл или сохранённый срез уже есть.

### Фильтры задач

Над таблицей можно быстро оставить:

- все задачи;
- готовые;
- задачи без файлов;
- устаревшие;
- задачи с ошибками;
- неполные.

Для одной задачи есть кнопка `повтор`. Она перезапускает только выбранную строку.

## 10. Что происходит при обработке данных

Обработка превращает разные выгрузки маркетплейсов в единый формат.

### Единые основные колонки

После стандартизации приложение приводит данные к таким базовым полям:

- `Дата`;
- `Маркетплейс`;
- `Категория`;
- `SKU`;
- `Бренд`;
- `Название`;
- `Продажи`;
- `Продавец`;
- `Средняя цена`;
- `Выручка`.

Если у маркетплейса колонка называется иначе, приложение приводит её к общему названию.

### Дата

В помесячной загрузке дата ставится как первый день месяца. Например, для апреля 2026:

```text
01.04.2026
```

Это нужно, чтобы все строки месячного отчёта имели понятную привязку к периоду.

### Бренд и продавец

Бренд и продавец приводятся к нижнему регистру и очищаются от лишних пробелов. Это помогает одинаково считать и искать значения, которые в исходных файлах отличаются только регистром.

### Продажи

Продажи приводятся к числу. Пробелы, неразрывные пробелы и запятые в числах очищаются.

## 11. Как рассчитывается вес и объём

Вес определяется из названия товара.

Приложение ищет в названии:

- граммы: `г`, `гр`, `g`, `грамм`;
- килограммы: `кг`, `kg`, `килограмм`;
- миллиграммы: `мг`, `mg`;
- миллилитры: `мл`, `ml`;
- литры: `л`, `l`, `литр`.

Для расчётов объёма жидкости условно переводятся в килограммы:

```text
1 л = 1 кг
1000 мл = 1 кг
```

### Упаковки и наборы

Если в названии указано несколько штук, приложение умножает количество на вес одной штуки.

Примеры, которые понимаются:

```text
3 x 175 г  -> 0.525 кг
175 г x 3 -> 0.525 кг
3 шт по 175 г -> 0.525 кг
3 шт - 175 г -> 0.525 кг
```

Если в названии найдено несколько весов, приложение выбирает наиболее вероятный итоговый вес: сумму упаковки, если она распознана, или максимальный одиночный вес.

### Контроль аномалий

После извлечения веса приложение проверяет значение.

Если вес не найден, равен нулю или выглядит некорректно, строка получает признак аномалии.

Если вес выше настройки `Максимальный вес, кг`, он тоже считается подозрительным. По умолчанию порог — `40 кг`.

В некоторых случаях приложение пытается исправить очевидный сдвиг в записи. Например, когда в названии литры или миллилитры записаны без десятичной точки. Если исправить безопасно не получается, вес оставляется пустым, а строка помечается как аномальная.

### Выходные колонки расчёта веса

**Вес, кг сырой**

Первое найденное значение до проверки.

**Вес, кг**

Итоговый вес после проверки и исправлений. Именно он используется в расчётах.

**Вес аномалия**

Признак, что вес нужно проверить вручную.

**Вес причина**

Короткое объяснение: всё хорошо, вес не найден, слишком большой вес, исправлен формат и т.д.

**Объем, кг**

Расчёт:

```text
Продажи, шт × Вес, кг
```

Если вес не найден, объём считается как `0`.

**Объем, т**

Расчёт:

```text
Объем, кг / 1000
```

**Цена за кг**

Расчёт:

```text
Средняя цена, руб / Вес, кг
```

Если вес не найден или равен нулю, цена за кг остаётся пустой.

## 12. Как работает склейка файлов

Склейка объединяет обработанные файлы в общий результат.

Перед сохранением приложение:

1. Приводит колонку продаж к числу.
2. Оставляет строки, где продажи больше `0`.
3. Убирает строки, где продажи `40000` или больше.
4. Убирает полностью одинаковые дубли строк.

Ограничение по продажам нужно, чтобы убрать пустые строки и явно подозрительные выбросы.

## 13. Как работает классификатор

Классификатор — это набор правил вида:

```text
если условие выполнено -> записать значение в нужную колонку
```

Пример:

```text
Если Название содержит "жидк"
то Подкатегория = "Жидкое"
```

Правила редактируются во вкладке `Классификатор` и сохраняются в:

```text
classifiers/rules.csv
```

Руками писать JSON не нужно: интерфейс собирает условия сам.

### Из чего состоит правило

**Правило активно**

Если выключено, правило сохраняется, но не применяется.

**Для категории**

Ограничивает правило конкретной категорией. Если стоит `*`, правило применяется ко всем категориям.

**Порядок**

Чем меньше число, тем раньше применяется правило. Если несколько правил подходят к одному товару, порядок особенно важен.

**Условия**

Где искать и что искать. Например:

- в `Название` содержит `жидк`;
- в `SKU` равно `123456789`;
- в `Бренд` не содержит `no name`.

Можно добавить несколько условий и связать их через `И` или `ИЛИ`.

Отдельный шаблон `если пусто` создаёт fallback-строку. Она читается так: если целевая колонка, например `Подкатегория`, всё ещё пустая после правил выше по порядку, записать указанное значение, например `Прочее`. Такая строка не ищет текст в `Название` или `SKU` и не должна оставлять пустыми классифицируемые колонки. Движок всегда применяет fallback только к пустым ячейкам целевой колонки, даже если в старом CSV для такого правила остался режим `перезаписать`.

**Куда записать**

Колонка, которую нужно заполнить или изменить. Например: `Подкатегория`, `Тип`, `Бренд`, `Вид мяса`.

**Что записать**

Значение, которое попадёт в выбранную колонку.

**Режим**

- `заполнить пустое` — значение записывается только там, где колонка пустая;
- `перезаписать` — значение записывается даже поверх уже заполненной колонки.

### Мясной справочник

Для категории `Мясо` колонка `Подкатегория` настроена по укрупнённому мясному справочнику. Правила пишут значения:

```text
Кулинария, Маринады, Крупнокусковые, Колбаски, Мелкокусковые,
Шашлык, Ребра крупнокусковые, Ребрышки маринадные,
Фарш, Порционные, Прочее
```

Если товар не попал в конкретное мясное правило, он получает `Прочее`. Более точные правила идут ниже по порядку и перезаписывают этот остаток: например, котлеты и биточки попадают в `Кулинария`, стейки/сатэ/каджун/маринады попадают в `Маринады`, шашлык попадает в `Шашлык`, а ребрышки в маринаде попадают в `Ребрышки маринадные`.

Для товаров с `Подкатегория = Колбаски` дополнительно заполняется колонка `Тип`. Возможные значения:

```text
Сосиски, Варёная колбаса, Сервелат (варёно-копчёная), Другое
```

Сосиски и сардельки попадают в `Сосиски`, варёные колбасы в `Варёная колбаса`, сервелат и варёно-копчёные позиции в `Сервелат (варёно-копчёная)`. Остальные колбаски, купаты, чевапчичи и люля-кебаб остаются в `Другое`.

### Быстрые шаблоны

Во вкладке `Классификатор` есть быстрые варианты:

- `по названию`;
- `по точному SKU`;
- `бренд из названия`;
- `если пусто`.

Они создают заготовку правила, которую остаётся заполнить.

### Внешний файл

Классификатор можно использовать отдельно от полной загрузки:

1. Откройте вкладку `Классификатор`.
2. В блоке `Внешний файл` выберите CSV или XLSX.
3. При необходимости включите `Сохранить копию XLSX`.
4. Нажмите `Обработать`.

Приложение создаст готовый CSV с применёнными правилами и покажет первые строки результата.

### Что меняется после классификации

В финальный classified-файл добавляется колонка `месяц` с короткой русской меткой месяца из `Дата`: `янв.`, `фев.`, ..., `дек.`. Она нужна для удобной фильтрации в сводных таблицах и попадает в куб/выгрузки как обычная колонка.

После классификации из финального пользовательского файла убираются служебные колонки контроля веса:

- `Вес, кг сырой`;
- `Вес аномалия`;
- `Вес причина`;
- `Объем, кг`.

Итоговый CSV при этом сохраняет пользовательские расчётные и классификационные колонки, включая `Объем, т`, `Цена за кг`, `Подкатегория`, `Тип`, `Вид мяса` и другие колонки, которые создали правила.

## 14. Раздел `Данные` -> `Файлы`

Здесь видны файлы текущего проекта.

Сверху есть фильтр по типу файла. У каждого типа есть подсказка: наведите курсор на значок рядом с названием типа.

**raw**

Исходные скачанные отчёты MPStats. Это резервный слой: если нужно заново обработать или классифицировать данные, raw обычно не надо скачивать повторно.

**processed**

Файлы после обработки: единые колонки, вес, объём, цена за кг.

**classified**

Файлы после применения классификатора. Именно они сохраняются в `Данные` -> `Куб`.

**Выгрузки**

Готовые XLSX из `Данные` -> `Выгрузка`.

Legacy `merged`-файлы в web-flow больше не показываются и не создаются автоматически: общий набор для работы собирается из `Данные` -> `Куб`, а финальные файлы создаются в `Данные` -> `Выгрузка`.

В таблице видно относительный путь, размер и дату обновления файла.

## 15. Раздел `Данные` -> `Куб`

Эта вкладка показывает, что уже сохранено в локальном кубе.

При сохранении среза в куб приложение отбрасывает строки без фактических продаж и объёма: в товарную БД попадают только позиции, где `Продажи, шт` больше `0` и доступная колонка объёма (`Объем, кг` или `Объем, т`) тоже больше `0`.

В таблице срезов видно:

- месяц;
- маркетплейс;
- категорию;
- количество строк;
- сколько дней месяца покрывает срез;
- файл-источник.

Колонка `Дней` показывает покрытие в формате `загружено/всего`. Для прошлых месяцев приложение считает месяц полным. Для текущего месяца дата актуальности берётся по дню, когда срез был сохранён в куб: например, если майская выгрузка сохранена 28 мая, в кубе будет `28/31 · по 28.05.2026`.

### Предпросмотр

Кнопка `Предпросмотр` показывает первые строки из таблицы товаров.

### Поиск

Поиск работает по основным товарным полям:

- `SKU`;
- `Артикул`;
- `Название`;
- `Бренд`;
- `Категория`.

Служебные поля БД в интерфейсе скрываются, чтобы не мешать просмотру.

### Как приложение защищается от дублей

При сохранении в БД приложение не должно бесконечно добавлять одни и те же строки. Для каждой строки считается внутренний отпечаток по проекту, месяцу, маркетплейсу, категории и содержимому строки.

Если тот же срез уже был сохранён, приложение пропускает дубли. Если включить `Перезаписывать БД`, старые записи для этой связки будут заменены.

Если нужно исправить ошибочные правила классификатора на уже загруженных данных, сохраните новые правила во вкладке `Классификатор`, откройте нужный план и нажмите `Переклассифицировать куб`. Для этого у задач должны сохраниться исходные processed-файлы; приложение заново создаст classified-файлы и заменит эти месячные срезы в БД.

После завершения переклассификации раздел `Данные` -> `Куб` перечитывает реестр и предпросмотр для текущего проекта. Если запуск ещё идёт, дождитесь статуса `готово`: до этого на экране может оставаться старый предпросмотр.

Кнопка с корзиной в строке среза удаляет registry-запись куба и строки товарной БД для связки `проект + месяц + маркетплейс + категория`. Исходные raw/processed/classified-файлы остаются на диске, поэтому срез можно собрать заново кнопкой `Собрать куб из готовых файлов`.

## 16. Раздел `Данные` -> `Отчёты`

Эта вкладка нужна для больших категорий и регулярных аналитических таблиц. Она строит агрегированные отчёты из куба в локальной DuckDB и отдаёт в Excel уже подготовленную таблицу.

Сырые строки из `mpstats_products` в интерфейс целиком не загружаются. Приложение считает отчёт SQL-запросом и показывает только первые строки предпросмотра.

Сверху видно:

- сколько категорий есть в кубе;
- сколько из них помечены как тяжёлые;
- сколько категорий выбрано для отчёта;
- текущий порог тяжёлого среза.

В списке `Тяжёлые категории` показаны категории, по которым лучше не открывать raw-данные в Excel. Для них доступны отчётные таблицы:

- `Категория по месяцам`;
- `Бренды по месяцам`;
- `Классификация по месяцам`;
- `Топ SKU`.

Порядок работы:

1. Откройте `Данные` -> `Отчёты`.
2. Выберите тип отчёта.
3. При необходимости сузьте период и категории.
4. Нажмите `Предпросмотр`.
5. Нажмите `XLSX` или `CSV`.

Если папку не менять, отчёты сохраняются в:

```text
data/projects/<название проекта>/reports/
```

После сборки в `Куб` обновляется время последнего отчёта для выбранных категорий.

## 17. Раздел `Данные` -> `Выгрузка`

Эта вкладка делает итоговые XLSX- или CSV-файлы из данных, уже сохранённых в локальной БД.

Сверху выбираются параметры:

- период;
- категории;
- нужные колонки;
- папка для результата;
- формат `XLSX` или `CSV`;
- режим `Разными файлами по категориям`.

### Шаблоны выгрузки

В блоке `Шаблон` можно сохранить текущие настройки выгрузки: период, категории, колонки, фильтры строк, сортировку, папку, формат и режим разбиения по категориям.

После сохранения шаблон появляется списком в этом же блоке:

- `Применить` подставляет настройки шаблона в форму выгрузки;
- `XLSX` или `CSV` сразу запускает выгрузку по шаблону из уже сохранённого куба;
- `Удалить` убирает шаблон.

Шаблон не скачивает MPStats заново. Он работает только с данными, которые уже есть в кубе текущего проекта.

Если папку не менять, файлы сохраняются в:

```text
data/projects/<название проекта>/exports/
```

Если указать свою папку, приложение автоматически добавит в неё подпапку текущего проекта. Например, для проекта `unit` путь `~/Reports/MPStats` превратится в `~/Reports/MPStats/unit`.

### Предпросмотр

Кнопка `Предпросмотр` показывает первые строки будущей выгрузки. В таблице можно:

- фильтровать значения по колонкам;
- сортировать по колонке;
- снять лишние колонки;
- убрать отдельные строки из текущей выгрузки.

В блоке `Фильтры строк` задаются фильтры, которые реально попадут в текущую выгрузку. Для числовых колонок можно выбрать условия `>`, `>=`, `<`, `<=`; например `Продажи, шт` `>` `0`, чтобы убрать строки без продаж.

Кнопка `убрать` не удаляет строку из БД. Она только исключает её из текущей спецификации выгрузки. Кнопка `Вернуть исключённые` очищает этот список.

После предпросмотра приложение показывает `Состав выгрузки`: конкретные сочетания `период / категория / маркетплейс / количество строк`. Этот список помогает увидеть, какие месячные срезы и категории попадут в итоговый файл.

### Экспорт XLSX/CSV

Переключатель `XLSX / CSV` выбирает формат результата. Кнопка выгрузки показывает выбранный формат.

Для `XLSX` в файле есть строка заголовков, заморозка шапки и Excel-фильтры по столбцам. Для `CSV` приложение создаёт CSV с разделителем `;` и кодировкой `utf-8-sig`, чтобы файл нормально открывался в Excel.

CSV-выгрузка больших наборов пишет файл напрямую из локальной БД, без промежуточной загрузки всех строк в память. Если в форме не выбрана сортировка, CSV сохраняется в естественном порядке БД; чтобы получить конкретный порядок строк, выберите колонку сортировки перед выгрузкой.

Если выключен режим `Разными файлами по категориям`, приложение делает общий файл по выбранным категориям. Если режим включён, для каждой категории/маркетплейса создаётся отдельный файл.

Excel ограничивает лист размером 1 048 576 строк. Приложение оставляет одну строку под заголовок, поэтому детальный raw-XLSX до 1 048 575 строк данных может быть выгружен обычным способом.

Если детальная XLSX-выгрузка превышает этот лимит, приложение не будет создавать набор тяжёлых raw-файлов. Для таких случаев используйте `Данные` -> `Отчёты`: Excel получит агрегированную таблицу для анализа и сводников.

Если файлов получается больше 10, приложение попросит подтвердить большую выгрузку.

Во время сборки появляется прогресс-бар: он показывает текущий шаг, процент, количество записанных строк и количество готовых файлов.

## 18. Раздел `Данные` -> `Качество`

Эта вкладка проверяет итоговые CSV прямо в интерфейсе. Она нужна перед передачей отчёта или большой XLSX-выгрузкой: экран быстро отвечает, можно ли доверять итоговому файлу.

### Как запустить проверку

1. Откройте `Данные` -> `Качество`.
2. Выберите проект в списке.
3. Нажмите `Проверить данные`.

Приложение проверяет итоговые classified CSV из web-проекта:

```text
data/projects/<project>/processed/
```

### Статусы

**OK**

Критичных проблем не найдено.

**WARNING**

Есть замечания: заметные пустые ключевые поля, неклассифицированные строки, аномалии веса/объёма или полностью одинаковые строки.

**FAIL**

Файл не найден, не читается или в нём 0 строк. Такой файл нельзя считать готовым.

### Что проверяется в MVP

Вкладка считает:

- общее количество строк;
- пустые ключевые поля: название товара, маркетплейс, категория и найденные классификационные колонки;
- покрытие веса/объёма: сколько строк имеют положительное значение веса или объёма;
- строки без веса/объёма и примеры таких товаров;
- аномалии веса/объёма: отрицательные, слишком большие и подозрительные значения; нулевой вес не считается аномалией;
- покрытие классификации: сколько строк классифицировано и сколько осталось пустыми;
- полные дубли строк: строка считается дублем только если совпадает по всем пользовательским колонкам. Повтор одного и того же `Артикул`/`SKU` в разных месяцах дублем не считается.

Если нужной колонки нет, проверка не падает и не портит статус сама по себе. В блоке `Не проверялось` будет указано, что именно не удалось проверить, например отсутствие колонок веса или классификации.

### Как читать результат

Сверху показан общий статус и какой источник проверен: `classified CSV` или `merged CSV`.

Карточки метрик показывают строки всего, покрытие веса/объёма, покрытие классификации, количество аномалий, полных дублей строк и пустых ключевых полей.

В таблице `Проблемы` видно тип проблемы, количество строк, долю и комментарий. Ниже есть блок `Примеры строк для проверки`: неклассифицированные товары, товары без веса/объёма, аномальные строки и полные дубли строк.

Кнопка `Сводка` копирует короткий текстовый итог проверки.

## 19. Где искать результаты на диске

Основные файлы web-проекта лежат внутри:

```text
data/projects/<название проекта>/
```

Внутри:

```text
raw/        исходные файлы MPStats
processed/ обработанные и классифицированные файлы
exports/   готовые XLSX/CSV-выгрузки
```

XLSX/CSV из `Данные` -> `Выгрузка` лежат в папке `exports` внутри проекта или в подпапке текущего проекта внутри папки, которую вы указали вручную.

## 20. Что проверять после запуска

После завершения загрузки проверьте:

1. Во вкладке `Умный план` нет задач со статусом `ошибка`.
2. В `Данные` -> `Куб` появились нужные месяцы, категории и маркетплейсы.
3. Количество строк выглядит правдоподобно.
4. В `Файлах` есть processed/classified файлы.
5. В `Данные` -> `Качество` статус `OK` или понятный `WARNING`, который вы готовы принять.
6. В итоговых данных заполнены нужные классификационные колонки.
7. У важных товаров корректно посчитаны `Вес, кг`, `Объем, т`, `Цена за кг`.
8. Строки с аномалиями веса проверены вручную, если они критичны для отчёта.
9. В `Данные` -> `Выгрузка` итоговый XLSX содержит нужные категории, период, строки и колонки.

## 21. Типовые проблемы и что делать

### Cookie устарел

Признаки:

- скачивание падает;
- MPStats отвечает ошибкой доступа;
- отчёты не создаются.

Что сделать:

1. Откройте MPStats в браузере.
2. Обновите cookie.
3. Вставьте новый cookie в приложение.
4. Нажмите `Сохранить настройки`.
5. Нажмите `Повторить ошибки`.

### Категория не появилась во вкладке `Категории`

Проверьте:

- строка сохранена в редакторе `Справочник`;
- включена `Активна в приложении`;
- указан маркетплейс;
- заполнен путь;
- период `От / До` не исключает выбранный месяц.

После правок нажмите `Сохранить CSV`, затем `Обновить справочник`.

### В плане меньше задач, чем ожидалось

Обычно причина одна из следующих:

- у части строк справочника выключена активность;
- период строки не пересекается с выбранным диапазоном;
- выбраны не все пути внутри группы;
- часть путей уже сохранена в БД и будет пропущена при запуске.

### Задача получила `нет данных`

Это значит, что для выбранного пути, фильтра и месяца MPStats вернул пустой результат. Проверьте путь, фильтр и период в MPStats.

### Ошибка на обработке

Чаще всего причина в неожиданном формате CSV или отсутствии нужных колонок в выгрузке. Откройте raw-файл через `Данные` -> `Файлы` и проверьте, что MPStats действительно отдал товарные колонки.

### Вес считается неправильно

Проверьте название товара. Если формат веса новый или нестандартный, строка может попасть в аномалии.

Для отчёта можно:

- проверить `Вес, кг`;
- посмотреть `Вес причина`;
- исправить правило парсинга в проекте, если таких товаров много;
- временно поправить итоговый файл вручную, если это единичный случай.

### Классификация не сработала

Проверьте правило:

- оно активно;
- категория совпадает или стоит `*`;
- условие ищет в правильной колонке;
- текст поиска действительно есть в товаре;
- режим `заполнить пустое` не мешает перезаписать уже заполненную колонку;
- порядок правил не приводит к тому, что более раннее правило заполняет колонку раньше.

Если нужно принудительно исправить уже заполненное значение, используйте режим `перезаписать`.

### В БД не появились новые строки

Проверьте:

- задача дошла до статуса `в БД`;
- выбран правильный проект;
- срез уже мог быть сохранён раньше;
- если нужно заменить старый срез, включите `Перезаписывать БД` и повторите задачу.

### В выгрузке нет строк

Проверьте:

- выбран правильный проект;
- данные уже сохранены в `Данные` -> `Куб`;
- выбранный период пересекается с сохранёнными месяцами;
- выбрана хотя бы одна категория;
- фильтры по колонкам не исключили все строки.

## 22. Безопасные правила работы

- Не отправляйте cookie в переписки и не сохраняйте его в публичные файлы.
- Не коммитьте локальную `mpstats.duckdb`, папку `data/` и рабочие CSV/XLSX-выгрузки.
- Для изменения категорий используйте CSV-справочник или редактор `Справочник`.
- Для изменения классификации используйте вкладку `Классификатор`.
- Перед повторной загрузкой большого периода сначала создайте план и проверьте количество задач.
- Не включайте `Перезаписывать БД`, если не уверены, что хотите заменить уже сохранённый срез.
- После изменения правил классификатора пересоберите нужные файлы или повторите классификацию.
- После изменения справочника создайте новый план: старый план уже содержит старый набор задач.
- Перед большой XLSX-выгрузкой проверьте оценку количества файлов в `Данные` -> `Выгрузка`.

## 23. Короткая шпаргалка

**Нужно скачать старые месяцы**

`Историческая загрузка` -> выбрать период -> выбрать категории -> `Создать план` -> `Запустить`.

**Нужно обновить следующий месяц**

`Ежемесячное обновление` -> `Синхронизировать новый месяц`.

**Нужно добавить категорию**

`Правила и справочник` -> `Справочник` -> `Добавить` -> заполнить строку -> `Сохранить CSV` -> `Категории`.

**Нужно поправить классификацию**

`Классификатор` -> выбрать/создать правило -> проверить условия и режим -> `Сохранить правила`.

Для уже сохранённых данных: открыть нужный план -> `Переклассифицировать куб`.

**Нужно повторить только ошибки**

`Повторить ошибки`.

**Нужно сохранить готовые файлы в куб без скачивания**

`Собрать куб из готовых файлов`.

**Нужно найти товар в сохранённых данных**

`Данные` -> `Куб` -> поиск по SKU, названию или бренду.

**Нужно сделать итоговый XLSX/CSV**

`Данные` -> `Выгрузка` -> выбрать период, категории и колонки -> `Предпросмотр` -> `Выгрузить`.

**Нужно быстро проверить доверие к итоговому файлу**

`Данные` -> `Качество` -> выбрать проект -> `Проверить данные`.
